# 权限点枚举与危险操作清单

> 本文档用于维护前后端统一权限点命名，并作为危险操作与审计日志中文动作映射的说明。
> 前端权限判断请优先使用 `canPerm("权限点")`，不要再零散使用 `role==='admin'`。

## 权限点命名规范

- 采用 `模块.动作` 形式，例如：`pcTx.delete`
- 前后端命名保持一致：
  - 前端：`src/utils/permissions.ts`
  - 后端：`functions/api/_permissions.ts`

## 常用权限点（当前已落地）

### 配件仓
- `tx.clear`：清空配件出入库明细（危险）
- `item.delete`：删除配件（危险）
- `stocktake.delete`：删除盘点单（危险）
- `stocktake.apply`：应用盘点（危险）
- `stocktake.rollback`：撤销盘点（危险）

### 电脑仓
- `pcAsset.update`：修改电脑台账
- `pcAsset.delete`：删除电脑台账（危险）
- `pcTx.delete`：删除电脑出入库明细（危险）
- `pcTx.clear`：清空电脑出入库明细（危险）

### 系统
- `audit.delete`：删除审计日志（危险）
- `audit.retention.manage`：管理审计保留策略（危险）
- `backup.run`：执行备份导出（危险）
- `restore.validate`：恢复前校验（危险）
- `restore.run`：执行恢复（危险）
- `restore.cancel`：取消恢复任务（危险）
- `user.manage`：用户管理（含删除用户）（危险）

## 危险操作与审计日志中文动作映射（自动）

危险操作清单已在 `src/utils/securityCatalog.ts` 中维护，`审计日志` 页面会自动合并该清单中的 `auditAction` / `auditEntity` 映射，避免后续新增危险操作时遗漏中文显示。

建议新增危险操作时同时补齐：
- `permission key`
- `auditAction`
- `auditEntity`
- 中文名称 `name`
